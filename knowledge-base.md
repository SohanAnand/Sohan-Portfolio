**What it does:**
BrandBrief AI is a natural-language template discovery system built for Canva Teams. It allows users to type a brief describing what they need (for example, "a LinkedIn post announcing our Series B that matches our brand guidelines") and receive brand-matched templates ranked by relevance with 72% accuracy in under three seconds.

BrandBrief AI bridges that gap using RAG, allowing users to describe what they need in natural language and letting the system do the semantic matching.

**Technologies:**

- RAG pipeline architecture

- Chose RAG over simpler alternatives (keyword search, usage-history recommendations, browsing UX redesign) because it most closely matched the user mental model of describing a need in natural language
- Implemented brand compliance scoring to ensure recommended templates were not just topically relevant but structurally close to what the user needed
- Used edit-distance measurement to minimize customization effort after template selection

**Results and Impact (Projected Model):**

- 40% projected reduction in design-team requests because users can self-serve the right templates

### AI and Technical Depth

**Q: How do you stay current with AI developments?**
A: I actively track AI research, model releases, framework updates, and emerging capabilities. I do not wait for features to be packaged into products. **Q: What is RAG and how have you used it?**
A: RAG stands for Retrieval-Augmented Generation. It is a technique that combines a language model with a vector search layer so the model can retrieve relevant information from a knowledge base before generating a response. In BrandBrief AI, I used RAG with Claude and pgvector to allow users to describe a design task in natural language and retrieve semantically matched brand templates. The retrieval layer converts the user's brief and the template library into vector embeddings, finds the closest matches, and passes them to the language model to rank and present with context. This approach worked better than keyword search because it matched on meaning and intent rather than exact terms.

**Q: What is multi-agent orchestration and when would you use it?**
A: Multi-agent orchestration is a design pattern where multiple AI agents collaborate, each handling a specific part of a complex task, with shared context passed between them. You use it when a problem is too complex or too long for a single model to handle well, or when different parts of a workflow benefit from specialized agent behavior. For example, in an IT incident management workflow like the ones I designed at Fabrix, one agent might detect anomalies, another might classify the incident, a third might generate a remediation recommendation, and a fourth might log the resolution and update the knowledge base. I understand the trade-offs between single-model and multi-agent approaches including reliability challenges, context management, and latency, and I design workflows that match the complexity of the problem rather than defaulting to the most complex architecture.

**Q: How do you evaluate whether an AI feature is working?**
A: I define evaluation criteria before the feature ships. For ML recommendations at Fabrix, I worked with the data science team to establish precision and recall thresholds that balanced recommendation accuracy against coverage, ensuring the system surfaced actionable insights without overwhelming users with false positives. For BrandBrief AI, the primary metric is retrieval accuracy (72% on the current prototype) and secondary metrics are time to first match and user adoption rate as a proxy for whether the system matches user mental models. AI features are not "done" when they ship. They need continuous measurement and tuning.

**Q: What is the difference between a good prompt and a bad prompt?**
A: A good prompt gives the model the context it needs, constrains the output format, specifies the reasoning approach, and accounts for edge cases. A bad prompt is vague, assumes the model has context it does not have, and produces outputs that require significant manual correction. I understand how language models process context and how token limits affect output quality. I use that understanding to structure prompts that produce reliable, actionable reasoning rather than generic responses. For AI product features, prompt design is as important as system architecture because the model's behavior is entirely a function of how you frame the task.
